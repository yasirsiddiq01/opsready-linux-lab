# Community and Pro feature boundary

The paid product must provide clear value beyond the free public edition.

| Capability | Community | Pro launch target |
|---|---:|---:|
| Guided command catalogue | 150 commands | 150 plus advanced modules and updates |
| Safe practice terminal | 25 commands, 10 attempts/reset | expanded command set and longer sessions |
| Incident scenarios | 50 | advanced incident packs and guided investigations |
| Assessment bank | 45 reviewed questions | expanded reviewed bank with saved history |
| Progress | browser session only | persistent account-based progress |
| Learning paths | introductory | structured multi-session paths |
| Results history | no | yes |
| Bookmarks and revision list | no | yes |
| Completion evidence | no | downloadable non-accredited completion record |
| Support | community feedback | priority issue and content support |
| Instructor features | no | later product tier |

## Launch rule

Do not charge for Pro until at least three meaningful paid-only capabilities are complete. The minimum recommended set is:

1. persistent accounts and progress;
2. expanded reviewed assessments;
3. structured Pro learning paths or advanced incident packs.
